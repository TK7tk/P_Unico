# 開発の文言
- 何度もやり取りをするのではなく、1度の作成で実際に使えるものにして下さい。
- 仕様上、事後的な要件変更で大きな影響を及ぼすと想定される事項を上位から5つ先に教えて

# IDE開発での以下は以下
‐ 簡単にサービスを作る場合は以下の技術要素で
    下記をもとにして、ローカルで動作確認するレベルでシンプルに実装を考えて
        - フロントエンド: Next.js (Pages Router) + TypeScript
        - バックエンド: ローカル
        - データベース: sqlite
        - スタイリング: Tailwind CSS + shadcn/ui + Radix
        - アイコン: Lucide Icons
    　　- アニメーション: Framer-motion
    Notローカル
 　　　　‐ 認証：Clerk
　　　　 ‐ DB：Supabase       
        - 決済：Straipe

# 開発含めた進め方
- まず作る/OUTPUTさせる → 即利用する → OUTPUT確認する → 修正する(チューニング) → ループ
‐ 沼ったら、「エラーハンドリングと生成プロセスの分解をしたいので、まず①××OUTPUTをワークフローで見たい」
‐ ワークフロー&フォルダの設計はマネジメントする上で大切。頭をしっかりと使うこと!!
- コツコツ / 一気に変えるのか？   
‐ AIツールはOpenAI/Googleが何をするのか？ を踏まえて実施することが大切
　そうでないと動画作成ツールのように作って満足に陥る!!
